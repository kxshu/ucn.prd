Workbench

定义

Workbench页面即系统首页，主要解决重要数据报表的呈现和任务的跟进。

相关模块如下：

模块名 备注
Quick Links
Mini Dashboards
System Task
Pending Task
Future Task
FYI
Push -

完整字段：

在线地址

需求说明

Quick Links

本模块主要让用户按个人喜好自行设置操作或者页面的快捷方式。

规则如下：

  1. 默认4个链接，分别为：
链接名 备注
Create Inquiry
Create Order
Create Logistic Order
Logistic Overview -

  1. 根据用户权限展示默认4个链接。

  2. 用户可以根据自行喜好添加更多的快速链接（权限范围以内），不设上限，且保留选择项目（下次登录仍然保留）。

  3. 可添加的快速链接范围包含：

    1. 所有的创建操作

    2. 所有可跳转的页面（注册、登录页面例外）

Mini Dashboards

本模块主要是方便用户快速查看各类数据的综合报表。

规则如下：

  1. 默认4个数据项，共计22可选数据项，默认4个数据为：Orders Placed（Purchase Orders模块）、Orders Delivered（Purchase Orders模块）、Orders In Process（Purchase Orders模块）、Orders In Process（Logistics Orders模块）。
数据项 模块项
Orders Placed Purchase Orders
Orders Delivered Purchase Orders
Orders In Process Purchase Orders
Orders In Process Logistics Orders
  1. 用户可以自行选择4个项，且保留选择项目（下次登录仍然保留）。
  2. 可以切换时间维度，可查看的数据项有：Yesterday、Recent 7days、Recent 15days、Recent 30days以及自定义时间段，默认查看截至于昨天的数据（Yesterday）。

System Task

本模块主要用于用户必要初始化系统。

Pending Task

本模块主要是方便用户查阅当前以及延期「我」的任务。

规则如下：

  1. 数据类型：Inquiry、Order、Logistic
  2. 数据字段，查看：在线地址
  3. 默认8条数据
  4. 支持搜索，字段排序，字段过滤、分页

Future Task

本模块主要是方便用户查阅未来需要「我」做的任务

规则如下：

  1. 数据类型：Inquiry、Order、Logistic
  2. 数据字段，查看：在线地址
  3. 默认8条数据
  4. 支持搜索，字段排序，字段过滤、分页

FYI

本模块主要是方便用户查阅仅需要「我」知道，并不需要「我」做的任务

规则如下：

  1. 数据类型：Inquiry、Order、Logistic
  2. 数据字段，查看：在线地址
  3. 默认8条数据
  4. 支持搜索，字段排序，字段过滤、分页

Push

本模块主要是方便用户查阅与「我」相关的，且需要「我」推进别人的任务

规则如下：

  1. 数据类型：Inquiry、Order、Logistic
  2. 数据字段，查看：在线地址
  3. 默认8条数据
  4. 支持搜索，字段排序，字段过滤、分页
