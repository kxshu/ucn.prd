PO Detail（以采购商端为例，供应商端参考，服务商端无）

定义

描述一个订单的详情页，Order进入可修改、双方确认以及可取消阶段

全局按钮说明：

操作按钮 被激活条件 点击后效果
Modify 当订单状态在TBC by Customer和Process时 按钮变为Send 和Cancle（取消修改，不保存修改）
Confirm 当订单状态在TBC by Customer 变为未激活
Copy 永远被激活状态 在新窗口创建新订单并复制此订单内容，注意不用带history
Cancle Order 当订单状态在TBC by Customer,TBC by Supplier和Process时 关闭此单，进入Cancled阶段
Create Order（供应商端无） 永远被激活状态 新页签跳转到Create Order页面
Mark As Important 与订单状态脱离，随时可被勾选 在己方的order overview标记
Highlight the Different, Hide the Same 待己方确认时，默认高亮不同 表格对应

供应商端参考采购商端，但将Custoemr文案改为Supplier

Basic info

本模块由采购商、供应商均可修改订单基本信息，字段见链接Order 字段/ po detail

规则如下：
  1. 同create order
  2. 当order处于需一方确认的阶段，默认高亮所有的修改项，但是供应商接单时不用高亮basic info板块

Responsibility

同create order

Payment

本模块显示财务记录，非线上支付模块

规则如下：
  1. 付款记录（注：供应商端文案为“收款记录”）由采购商发起，供应商或服务商需做“确认”和可做“提醒采购商付款”（根据款项名称判断发送给不同提醒对象）
  2. 采购商可对自己的付款记录操作“无效”，无效的付款记录不计入Total
  3. 采购商可对自己的付款记录操作“修改”，不论该状态是否为“已确认”
  4. 退款记录由供应商发起，采购商需做“确认”和可做“提醒供应商退款”

Product Info

本模块显示该order内的所有产品信息，采购商和供应商可操作增、删产品，并对每个SKU详情做修改和备注Product Details。此外，采购商还可以可勾选部分产品完成在之后流程中会出现的其他快捷操作，如创建QC单、创建物流单。

规则如下：
  1. Add product从平台的产品库添加产品，输入搜索条件点击search触发，clear为清空搜索条件
  2. Add product从收藏夹添加产品，同上
  3. Add product从Inquiry添加产品，通过Inquiry No., SKU Code精准搜索
  4. Remove从表中删除该产品
  5. 每个产品行分为第一排在平台产品库中的公共信息，第二排Demand为定制化信息，采购商可在此行和remark行填写内容。注：有部分字段为下拉选择；字段中价格部分是三选一，例如FOB有填写，则EXW部分不能填写
  6. Total行对可计算的字段汇总
  7. Detail进入平台产品库中该产品详情页
  8. Create QC Order勾选相应产品创建QC Order，条件是该产品已入库。同一SKU不能分批QC，但是可以多次建QC，且系统提示某些SKU已有QC Order
  9. Place Logistics Plan勾选相应产品创建物流计划。系统判断：如果已勾选产品之前被创建过物流计划，那么默认不带入
  10. 申请返工勾选相应产品返工，默认整条SKU返工
  11. 申请退货勾选相应产品返工，默认整条SKU退货
