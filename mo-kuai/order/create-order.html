Create Order（采购商端，供应商端无，服务商端无）

定义

采购商发起一个新订单

全局说明：

  1. 进入时自动创建一个新的订单编号并自动填充
  2. Send提交给指定供应商
  3. Save As Draft存进草稿箱中并返回Order Overview界面
  4. Quick Create弹出Inquiry Overview 弹出框，会覆盖已填写basic info 部分，不会清空product info已添商品
  5. 若没有保存也没有提交，则系统不记录此单，该订单编号也自动释放

Quick Create

本模块由Quick Create按钮触发，承上一流程Inquiry，可将某一Inquiry中basic info, product info快速填充至该Order的相应模块

规则如下：
  1. 弹出面板信息为所有已完成的Inquiry
  2. 可输入Inquiry No.或者SKU Code精准模糊搜索Inquiry内容
  3. 在搜索结果中选对应Inquiry快速创建并同步Basic Info中的“参考Inquiry No”字段

Basic info

本模块由采购商填写订单基本信息，字段见链接Order 字段/ po detail之外，还有Order Remark ,Attachment

规则如下：
  1. 字段表中标黄部分为必填项
  2. Order Remark为输入文本框，大小固定，超出文本用滚动条辅助显示
  3. Attachment显示框大小固定，滚动条辅助。旁边为上传按钮。
  4. 点击任意附件显示下载提示框。
  5. 附件旁有删除按钮，点击显示提示框。

Responsibility

用于约定买卖双方的特点素材提供节点，一旦设定将定期收到系统提示且不可更改。
当实际完成时需在相应列添加实际完成时间。

规则如下：
  1. responsibility的系统提醒时间：
    1. 如果提前于计划完成，提醒时间不做变动；
    2. 如果延期完成，提醒时间随计划间隔时间顺延
  2. 自己只能设定自己的素材提供时间

Product Info

本模块显示该order内的所有产品信息，采购商可根据需要增加和删除，并对每个SKU详情做修改和备注

规则如下：
  1. Add product从平台的产品库添加产品，输入搜索条件点击search触发，clear为清空搜索条件
  2. Add product从收藏夹添加产品，同上
  3. Add product从Inquiry添加产品，通过Inquiry No., SKU Code精准搜索
  4. Remove从表中删除该产品
  5. 每个产品行分为第一排在平台产品库中的公共信息，第二排Negotiation为定制化信息，采购商可在此行和remark行填写内容。注：有部分字段为下拉选择；字段中价格部分是三选一，例如FOB有填写，则EXW部分不能填写
  6. Total行对可计算的字段汇总
  7. Detail进入平台产品库中该产品详情页
