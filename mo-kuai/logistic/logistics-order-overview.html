Logistics Order Overview（以采购商端为例，供应商端参考，服务商端无）

定义

Logistics Order Overview页面即对历史物流订单的总览，主要解决物流订单进度跟踪和物流订单查找功能，并可切换浏览方式订单、集装箱和产品

相关模块如下：

板块 备注
顶部导航 二级菜单：Logistics Plan Overview、Logistics Order Overview
侧边栏
状态栏及对应的操作
搜索栏及对应的浏览方式和明细

Logistics字段

需求说明

侧边栏

  1. 第一部分：Quick Link

    默认Inquiry Overview、Order Overview、Warehouse Overview、Logistics Plan Overview，以及Add Link，每个快捷链接可移除

  2. 第二部分

    1. Recycle Bin（？）：？代表Logistics Order模块的回收数，点击新页签打开

    2. Logs：记录有数据变动的操作：创建和删除物流订单

  3. Actions

    Place Logistics Plan：跳转到Place Logistics Plan页面

    Place Logistics Order：跳转到PLace Logistics Order页面

    Download the Overview ：所有Logistics Order不加筛选条件导出，弹出下载面板，导出模板见Logistics导出模板

搜索栏及对应的浏览方式和明细

本模块根据用户的搜索内容、浏览方式以及选择的当前物流单阶段，判断明细表呈现的维度

规则如下：
  1. 搜索方式：Logistics Plan No，Logistics Order No，Container No，Order No., SKU Code
  2. 浏览方式：Logistics Plan/Order, Transport Unit，SKU（view by 字段）
  3. 当通过Logistics Plan No和Logistics Order No搜索，
    1. 默认view by Plan/Order，字段以描述物流单为主，但根据集装箱分行。注：切换到该物流单对应状态栏；
    2. 当切换到view by transport unit，行数不变，字段名有变化。注：切换到该物流单对应状态栏；
    3. 当切换到view by sku，字段以描述SKU为主，且显示该物流单包含的所有sku行、再根据集装箱分行。注：状态栏不做切换
  4. 通过Container No搜索，
    1. 默认view by transport unit，数据只有一行
    2. 当切换到view by Plan/order，数据只有一行
    3. 当切换到view by sku，以SKU拆分行
  5. 通过sku号搜索，
