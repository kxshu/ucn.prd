Place Logistics Order（采购商端，供应商端无，服务商端无）

定义

采购商发起一个新物流订单

全局说明：

  1. 进入时自动创建一个新的物流订单编号并自动填充
  2. Send提交给指定供应商
  3. Save As Plan存为Logistics Plan并返回Logistics Plan Overview界面
  4. Quick Create弹出Logistics Plan Overview 弹出框，会覆盖已填写basic info 部分，不会清空product info已添商品
  5. 若没有保存也没有提交，则系统不记录此单，该订单编号也自动释放

相关模块如下：

Quick Create

本模块由Quick Create按钮触发，弹出Logistics Plan Overview弹出框，勾选某一Logistics Plan复用

规则如下：
  1. 弹出面板信息为所有的Logistics Plan Overview
  2. 可输入Logistics Plan No.或者SKU Code模糊搜索物流计划单
  3. 在搜索结果中选对应Logistics Plan 快速创建，将信息快速填充进Logistics Order，Container Info和Product Info部分累加，Basic Info部分覆盖

Basic info

本模块部分由采购商填写物流订单基本信息，字段见链接Logistics\basic info字段

Transport Info

本模块部分由采购商填写物流订单运输信息，字段见链接Logistics\transport info字段

Container Info

本模块为集装箱信息，由采购商填写。注：Detail页面会增加其他信息列，由供应商填写

规则如下：
  1. Add Container增加一行集装箱信息
  2. Remove Container删除勾选行
  3. 货柜类型列为下拉字段
  4. 货柜数量列为填写数字字段

Product Info

本模块显示该order内的所有产品信息，采购商可根据需要增加和删除，并对每个SKU详情做修改和备注

规则如下：
  1. Add product从仓库添加产品，输入搜索条件：SKU Code、Order No和Inbound No，模糊搜索，点击search触发
  2. Remove从表中删除该产品
  3. Total行对可计算的字段汇总，需汇总信息见字段表Logistics\product info\total行 字段
  4. Detail进入平台产品库中该产品详情页
