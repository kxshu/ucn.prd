Sign

定义

用户录入相关信息在系统注册账号，通过用户名和密码登陆系统。

模块说明

Sign in

1.行为规则

用户录入Email和Password，点击Log in登陆系统，系统使用Email和Password验证数据库是否存在对应的数据，若存在，登陆成功跳转对应的workbench页面（采购商、供应商、服务商），若失败返回提示信息：Email或Password有误，请检查后重新输入！

2.业务规则

1.Password输入在页面显示为：********

2.取消remenber me ，页面自动记住用户名和密码，不记住登陆状态

3.forget password链接到forget password页面

4.No account? Sign up now>>链接到Sign up页面

3.数据规则

Email和Password直接使用Sign up的User Information里面的Email和Password字段

4.原型页面

https://axhub.im/pro/282c989ac491f80f/#g=1&p=1_1-sign_in

Sign up

1.行为规则

1.录入User Information和Company Information的表单信息，勾选I have readed "xxx"，点击OK，系统自动为用户注册好账号，用户名即用户录入的Email地址。

2.注册成功，系统提示注册成功并跳转对应的workbench页面（采购商、供应商、服务商）。

2.业务规则

1.未勾选I have readed "xxx"，OK按钮置灰不可点击，勾选后OK按钮恢复正常

2.点击I have readed "xxx"，系统跳转 "xxx"协议页面，显示具体的协议内容

3.点击Go Submit a Request>>，跳转到Request to Get Invitation Code页面

4.录入Invitation Code后，系统自动复用Request to Get Invitation Code的相关字段信息覆盖sign up的Company Information字段信息进行显示

5.必填项未填写完整就点击OK按钮，系统高亮显示未填写的必填项

6.填入Email，系统实时检测该地址是否已存在，若存在立即提示：该Email已存在，请换一个Email！

7.不同端口（采购商、供应商、服务商）注册后该Email记得要能识别到对应的终端，不能出现在采购商注册的Email登陆到供应商终端的问题

3.数据规则

User Information
字段 是否必填 默认值 类型 备注
Invitation code 是 varchar 2
User Name 是 varchar 2
Email 是 varchar 2
Password 是 varchar 2
tel varchar 2
Company Information
字段 是否必填 默认值 类型 备注
Company Name 是 varchar 2
Company Type 是 varchar 2 字典项待提供
Website varchar 2
Country 字典 获取国家项
City 字典 获取城市项，与国家联动
Adderss varchar 2
Contact Number 是 varchar 2

4.原型页面

http://120.79.230.214/servicer/#g=1&p=1_3-sign_up

Request to Get Invitation Code

1.行为规则

1.用户录入公司相关信息，提交系统

2.系统自动生成邀请码，返回信息：提交成功，我们将尽快联系您！

2.业务规则

1.邀请码生成规则待定

2.必填项未填写完整就点击Submit，系统高亮显示未填写的必填项

3.数据规则

Company Information

Website varchar 2
Country 字典 获取国家项

Foget the password

1.行为说明

1.在Input email页面录入Email，点击Submit

2.系统跳转到Identify页面并自动向该email发送一个链接，与此同时页面显示出重新发送按钮，用户点击后系统再次向该email发送链接

3.用户点击邮箱的链接后系统自动在新标签打开Reset password页面，用户录入新密码，点击Submit，系统跳转Finish页面

4.用户在Finish页面点击Go sign in now>>，链接到Sign in页面

2.业务规则

1.必填项未填写完整就点击Submit，系统高亮显示未填写的必填项

2.若录入的email在我们系统不存在，返回：您输入的emai未注册，请核对后重新输入！

3.若用户在Reset password页面两次输入的密码不一致提示：两次输入的密码不一致，请核对后重新输入！

3.数据规则

暂无
