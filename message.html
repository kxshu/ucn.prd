Message

定义

平台消息包括系统消息和业务消息，系统消息是平台管理人员手动录入消息内容向所有平台使用者发布的消息；业务消息是根据业务规则，向采购商、供应商、服务商分别发布的关于业务数据提醒的相关消息。消息有两部分显示方式，一种是系统将消息发送到Email，另一种是通过站内信的方式显示，站内信包括页面右上角全局显示的系统通知消息和workbench显示的业务数据提醒消息。

模块说明

Message-System Message

1.行为规则

1.用户进入页面，系统默认按消息发布时间倒序显示

2.勾选未读的消息（可多选），点击Mark as Read，系统将该消息置为已读

3.用户在搜索框内输入关键字，点击查询图标或回车，系统根据关键字在消息Content内进行检索返回

2.业务规则

1.message management按钮链接到message management页面，此按钮只有平台系统管理人员可见

2.未读的消息前面显示黄色的点，消息已读后该消息前面的黄点消失

3.消息内容直接在消息列表进行显示，字数超长的自动换行显示

4.进入此页面时系统默认显示System Message的页签内容

3.数据规则

暂无

Message-Message Settings

1.行为规则

1.用户点击Message Settings页签，系统显示Message Settings的页签内容

2.用户在此页面选择勾选系统消息的发送方式，可进行的操作为勾选或取消勾选

2.业务规则

1.勾选Platform，系统通过站内信的方式进行发送消息；勾选Email，系统通过邮件来发送消息

2.Platform默认处于勾选状态，且置灰不可取消勾选；邮件默认置于未勾选状态，用户可进行勾选

3.消息类型包括：System Message，Pending Task，Future Task，FYI，Push

3.数据规则

暂无

Message management

1.行为规则

1.用户进入页面，系统默认按消息保存时间降序排列

2.用户点击新增消息按钮，系统弹出消息编辑弹窗，用户录入信息，可点击<发布>/<保存>/<取消>

3.点击发布后，系统将给消息发送给系统所有用户并关闭弹窗，将消息保存到消息管理列表；点击取消，系统关闭弹窗，不保存此次数据

4.用户在消息管理列表可对没有发布的消息进行修改，删除和发布操作

2.业务规则

1.新增消息时点击保存，系统将此消息保存到消息管理列表，不发送给平台用户

2.消息未发布状态，用户可对此消息进行删除、修改和发布操作，消息发布之后该消息不能再进行任何操作，操作链接全部不可见

3.数据规则

Message
字段 是否必填 默认值 类型 备注
ID 是 varchar 2 系统自动生成
Content 是 varchar 2 多行文本
Time date 精确到秒
